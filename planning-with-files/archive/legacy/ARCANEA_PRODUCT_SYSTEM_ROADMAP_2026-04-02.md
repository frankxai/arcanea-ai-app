# Arcanea Product System Roadmap — 2026-04-02

## Goal

Build Arcanea into a unified creative intelligence operating layer that combines:
- chat
- notes
- whiteboard
- generation workflows
- project graph
- memory
- provenance
- creator social identity
- agent crews
- BYOK-first provider power

## Product Thesis

Arcanea should own the work, not just the prompt.

The product is not:
- another chat app
- another prompt wrapper
- another whiteboard
- another note app

The product is:
- a creative intelligence workspace with a compounding project graph

## Core Product Objects

- workspace

Every major surface should map back into these objects.

## Build Sequence

## Phase 1 — Finish Graph Foundation

Scope:
- live Supabase activation
- real type regeneration
- project-aware retrieval on live data
- async graph enrichment
- provider routing and usage tracing

Why:
- this makes Arcanea coherent before more surfaces are added

## Phase 2 — Notes / Docs

Scope:
- first-class docs linked to projects
- notes saved as graph objects
- docs visible to retrieval and agents
- doc provenance and versioning

Acceptance:
- users can create, organize, retrieve, and AI-work from project docs

## Phase 3 — Board / Whiteboard

Scope:
- first-party lightweight board
- notes, images, links, AI cards, connectors
- board linked to project graph

Acceptance:
- users can visually organize ideas without leaving the Arcanea project

## Phase 4 — Capture / Extension

Scope:
- browser extension
- import outputs from other AI tools
- project assignment on ingest
- provenance capture
- summary/tag extraction

Acceptance:
- Arcanea becomes the place where external AI work is absorbed and organized

## Phase 5 — Agent Crews

Scope:
- style-aware agents
- lore architect
- canon editor
- content curator
- coding agent
- social strategist

Acceptance:
- agents operate on the user’s graph, not generic blank context

## Phase 6 — Social Compounding Layer

Scope:
- creator profiles
- collections
- prompt books
- follows
- challenge reputation
- remix/provenance graph

Acceptance:
- Arcanea compounds through creator identity and artifact networks

## Monetization Strategy

## Moat

The moat is not one feature.

The moat is:
- continuity
- graph memory
- provenance
- style learning
- creator identity
- social artifact compounding
- open ecosystem with Arcanea-native taste

## Execution Principles

- custom product model, borrowed subsystems
- do not replatform around generic agent frameworks
- keep BYOK-first now
- keep retrieval and graph quality high
- let docs, board, and studio all feed the same graph
