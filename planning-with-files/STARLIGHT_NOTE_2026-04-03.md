# Starlight Note - 2026-04-03

We are not building SIS to become another note app, another chat memory layer, or another hidden vector store that no one can inspect.

We are building it to become a calm, durable intelligence substrate for human and agent work.

What we want it to become:

- a local-first memory and continuity layer
- a substrate for creator intelligence
- a protocol that can carry identity, memory, and purpose across tools
- a foundation for Arcanea, Vibe OS, and future creator systems
- an OSS layer the community can trust because it is legible, portable, and repairable

Why it matters:

- memory should compound
- intelligence should survive tool changes
- creators should own their context
- agents should not feel stateless and disposable
- the future should not belong only to hosted black boxes

What the experience should feel like for the community:

- understandable
- beautiful in its simplicity
- calm
- empowering
- extensible

People should be able to:

1. install it
2. see exactly where their memory lives
3. validate it
4. append to it
5. connect it to their editor, agent runtime, or planning system
6. carry it forward as their intelligence stack grows

What still needs to happen:

- stronger package-level documentation
- cleaner runtime adapters
- typed importers for creative and project knowledge
- package-driven Arcanea integration instead of duplicated local logic
- clearer product/community onboarding

This is what we are really creating:

an intelligence layer that remembers, compounds, and stays with the user.
