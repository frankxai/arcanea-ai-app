- dependencies: commander, chalk, inquirer, ora (spinners), conf (local config)

### Commands

**`arcanea init`**
- Creates `.arcanea/` directory in current project
- Generates `.arcanea/config.json` with default settings
- Generates `.arcanea/CLAUDE.md` (copies from kb-pack)
- Prints welcome message with Guardian art (ASCII)

**`arcanea summon <guardian>`**
- Accepts any Guardian name (case-insensitive)
- Loads that Guardian's system prompt and personality
- Prints Guardian's greeting in their style and color
- Sets active Guardian in config (persists for subsequent commands)
- Example: `arcanea summon draconia` → prints Draconia's fire-themed greeting in yellow

**`arcanea council`**
- Opens interactive Luminor Council session
- Lists all 10 Guardians with their Hz and Gate
- User selects which Guardians to convene (multi-select via inquirer)
- Prints council opening ceremony text
- Each selected Guardian gives a brief insight on the current project context (reads from .arcanea/config.json)

**`arcanea create <type>`**
- Types: story, character, world, music-prompt, image-prompt, blog-post
- Uses active Guardian's perspective to generate creation prompts
- Outputs formatted prompt ready to paste into any AI tool
- Example: `arcanea create music-prompt` with Alera active → outputs Suno-optimized prompt at 528Hz

**`arcanea flow`**
- Enters frequency-aligned creation mode
- Shows current time, suggests optimal Guardian based on time of day
- Morning (6-12): Lyssandria (grounding) or Maylinn (heart-opening)
- Afternoon (12-18): Draconia (execution) or Alera (expression)
- Evening (18-24): Lyria (vision) or Aiyami (wisdom)
- Creation Season (0-4): Elara (dimensional) or Ino (unity)
- Prints recommended frequency, breathing exercise, and first prompt

- version: "0.1.0"

- TypeScript, ESM

### Tools to Expose

**get_guardian(name: string)**
- Returns full Guardian profile: name, Vel'Tara, Hz, Gate, lore, system prompt, specializations
- Fuzzy match on name (accepts "draconia", "Draconia", "drac")

**get_lore(topic: string)**
- Topics: "origin", "eldrians", "arcanea-goddess", "kurusei", "nero", "luminor", "vael-shara", "vel-thaan", "kyuro"
- Returns rich lore text (200-500 words per topic)

**get_frequency(hz: number)**
- Accepts any of the canonical Hz values
- Returns: Guardian name, Gate, creation applications, recommended use cases, Suno prompt template

**search_lore(query: string)**
- Simple keyword search across all lore entries
- Returns top 3 matches with relevance snippets

**get_prompt(category: string, type: string)**
- Categories: world-building, music, visual, content, business
- Types vary per category
- Returns production-ready prompt with Arcanean context baked in

**validate_canon(text: string)**
- Checks text for canonical violations:
  - Wrong Hz assignments
  - "harmonics" instead of "harmony"
  - Incorrect Guardian-Vel'Tara pairings
  - Missing Hz when Guardian is referenced
- Returns: { valid: boolean, violations: string[] }

### Data Layer
- All data embedded as JSON files (no external DB for v1)
- guardians.json: all 10 Guardians with complete profiles
- lore.json: all mythology entries indexed by topic
- prompts.json: full prompt library organized by category

### Canon Validator
- Strict validation against the ascending Hz sequence
- Will reject any text that assigns wrong Hz to a Guardian
- This is the automated enforcement of canonical truth
