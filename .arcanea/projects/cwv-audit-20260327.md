## Top Opportunities (All Pages)

### 1. Avoid Multiple Page Redirects (Critical)
- **Savings:** 790-960ms across all pages
- **Impact:** Every page is hit by redirect chains. Likely `arcanea.ai` -> `www.arcanea.ai` or HTTP -> HTTPS redirects.
- **Fix:** Ensure the canonical URL is used everywhere (no redirect hops).

### 2. Reduce Unused JavaScript (Critical)
- **Homepage mobile:** 81 KiB savings
- **/chat mobile:** 257 KiB savings (worst offender)
- **/imagine mobile:** 145 KiB savings
- **Fix:** Code-split aggressively. Dynamic import heavy modules (chat SDK, image generation, Three.js). Tree-shake unused exports.

### 3. Reduce Unused CSS (Moderate)
- **Savings:** ~38-39 KiB across all pages
- **Fix:** Purge unused Tailwind classes. Consider critical CSS extraction.

### 4. Minimize Main-Thread Work (Critical)
- All pages score 0 on this diagnostic
- **Fix:** Move heavy computation to Web Workers. Defer non-essential JS. Reduce component hydration cost.

### 5. Reduce JavaScript Execution Time (Critical)
- All pages score 0
- **Fix:** Code-split, lazy load, defer. The Three.js lazy load optimization (not yet deployed) should help significantly.

### 6. Avoid Large Layout Shifts (Homepage + Chat)
- Homepage CLS: 0.205, Chat CLS: 0.563
- **Fix:** Set explicit width/height on images and embeds. Reserve space for dynamic content. The image dimensions fix (not yet deployed) should help.

### 7. Render Blocking Requests
- All pages affected
- **Fix:** Defer non-critical CSS/JS. Inline critical CSS. Use `font-display: swap` for web fonts.

The chat page loads 257 KiB of unused JS. Dynamic import the chat SDK, AI streaming, and tool-result rendering on demand.

### P3 - Critical CSS
Extract above-the-fold CSS and inline it. Defer the full Tailwind stylesheet.

### P4 - LCP Optimization
The LCP element (likely the hero section or a large image) takes 11-15s to paint. Investigate:
- Is the LCP element an image? Use `priority` / `fetchpriority="high"` + preload.
- Is it text rendered after font load? Use `font-display: swap` + preload fonts.
- Is it blocked by JS hydration? Consider SSR/streaming for the critical content.

### P5 - Web Workers
Move any heavy computation (markdown parsing, syntax highlighting, image processing) off the main thread.
