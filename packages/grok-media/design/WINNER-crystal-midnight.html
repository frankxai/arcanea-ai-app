Crystal Midnight — Design Notes

The winning hybrid. Every design decision documented with its source variation.

Design Philosophy

Crystal Midnight synthesizes three competing philosophies: V04 Midnight Minimal's structural discipline and performance obsession, V01 Crystal Glass's brand identity and visual signature, and V03 Aurora Flow's emotional warmth and living personality. The result is a design that feels like a professional tool (V04), looks unmistakably like Arcanea (V01), and breathes with just enough life to create genuine delight (V03).

The guiding principle: restraint is the default; every embellishment must earn its place. The animated border, the breathing orb, the glowing progress dot — each is a single, intentional moment of magic in an otherwise disciplined interface.

Color System

Eight neutral values from V04 form the foundation. Three accent colors from V01 provide brand identity. The system is deliberately narrow — no color should appear without a clear functional role.

Typography

V04 Inter with -0.02em letter-spacing throughout. Weights restricted to 400 (body), 500 (labels/buttons), and 600 (headings/brand). Size scale: 9px (shortcut hints, badges), 10px (footer, compact text), 11px (metadata, filters, stats), 12px (body, filenames, settings), 13px (panel title). Nothing exceeds 13px — density over drama. font-variant-numeric: tabular-nums on all numeric displays for aligned, professional data presentation.

Layout System

  • V04 Side Panel: 400px wide, 700px tall — optimal for Chrome Extension side panel at standard viewport heights
  • V04 Popup: 280px wide, 440px tall — compact toolbar popup format
  • V04 4px grid alignment — all padding, margins, and element sizes are multiples of 4px. Exceptions allowed only for 1px borders and 2px progress bars.
  • V04 2px media grid gap — creates the signature dense mosaic where content touches. Images breathe through their own color, not through whitespace.
  • V01 Masonry support — grid-row: span 2 on tall items breaks the uniform grid and accommodates varied aspect ratios naturally.

Component Decisions

  • V04 Tab underline indicator (not segmented control) — cleaner, uses 1px line that matches border system. Tinted teal instead of V04's white for brand consistency.
  • V04 Keyboard shortcut hints — [1] [2] [3] on tabs, [S] [D] on action buttons. Small grey badges that are visible but subordinate. Power-user friendly without cluttering. No other media extension offers this.
  • V04 Ghost buttons by default — transparent background + subtle border. Solid teal reserved ONLY for the primary CTA (Download). This creates unambiguous action hierarchy.
  • V04 Filter pills — All / Image / Video / HD / SD. Simple text toggles with background fill on active. No borders, no icons, no badges — text is enough.
  • V01 Animated conic gradient border — The signature visual element. A single teal-to-violet gradient sweeps around the outer panel frame at 6s rotation, 0.25 opacity. Uses CSS @property for smooth angle interpolation and mask-composite: exclude for the border-only effect. Applied only to the outermost frame — never on inner cards or elements.
  • V01 Brand icon — Layered hexagonal diamond SVG with teal-to-violet linear gradient. Three nested hexagons create depth. Center circle provides a focal point. Instantly recognizable as Arcanea.
  • V01 Video badges in violet — #8b5cf6 background distinguishes video content from images at a glance. HD badges use gold (#ffd700). Both are 9px text, 3px border-radius, tight padding.
  • V02 Selection counter showing ORDER — Selected items display 01, 02, 03, 04 instead of a generic checkmark. This adds information value (the user knows download order) and creates a sense of intentional curation. 18px teal square with 9px tabular-nums text.
  • V03 FAB with ambient pulse — Teal gradient button in the bottom-right corner with two concentric expanding ring animations staggered by 1.5s. Creates a subtle "heartbeat" effect that draws the eye without demanding attention.
  • V03 Breathing empty state — A central teal orb pulsates with a slow 3s breathe cycle while a single ring expands outward and fades. Single-color teal (not V03's four-color aurora) to match Crystal Midnight's restrained palette.
  • V03 Dual-ring scanner — Outer ring at 3s rotation, inner ring at 2s counter-rotation. Different speeds create an organic, non-mechanical feeling. Center dot pulses. Reduced from V03's triple-ring for performance.
  • V03 Progress bar glow dot — Active download progress bars have a 6px glowing dot at the leading edge with a pulsing box-shadow. Makes downloads feel alive and in-progress rather than static.

Performance Profile

Crystal Midnight is designed for Chrome Extension constraints where the panel runs alongside complex web pages (grok.com, x.com) that are already GPU-intensive.

  • No backdrop-filter anywhere — the single most expensive CSS property in extension contexts.
  • One web font — Inter at three weights (400, 500, 600). Inter is also a system-common font, so the @import is a progressive enhancement, not a hard dependency.
  • One continuous animation in library view — the border sweep. Uses CSS @property which is GPU-composited. The FAB pulse is lightweight (opacity + transform only).
  • Zero JavaScript animations — all motion is pure CSS. The stagger effect on grid items uses CSS animation-delay, not JS IntersectionObserver.
  • Scanning and empty states are temporary views — their animations only run when active, never in the background.
  • Estimated CSS weight: ~12KB unminified. ~4KB minified + gzipped. Well within extension performance budgets.

What Crystal Midnight Does NOT Include

Deliberate omissions are as important as deliberate inclusions:

  • No backdrop-filter / glassmorphism — the Design Knowledge Base says "Glassmorphism is dead." V01's glass layers were its biggest performance risk.
  • No aurora/four-color gradients — V03's four-color system (teal, blue, purple, pink) created visual richness but reduced information hierarchy. Crystal Midnight uses teal as the single active accent with violet restricted to badges and the border gradient.
  • No floating blobs — V03's three animated background blobs were ambient but GPU-heavy. Cut entirely.
  • No gradient text — V01's gradient brand name was striking but adds complexity. Plain white text is clearer and more legible at small sizes.
  • No custom fonts beyond Inter — V03's DM Sans was excellent but adds a font dependency. Inter covers all needs and is near-ubiquitous as a system font.
  • No 3D transforms — V07's perspective system was impressive but inappropriate for an extension. Flat hierarchy with color-based depth instead.

Implementation Notes

  • The @property declaration for --gradient-angle is required for the animated border. Supported in Chrome 85+ (the extension's only target). Fallback: static subtle border.
  • The mask-composite: exclude technique creates the border-only effect without an extra wrapper element. Both -webkit-mask-composite: xor and mask-composite: exclude are declared for compatibility.
  • Grid stagger animation uses animation-fill-mode: both via the both shorthand to prevent a flash of unstyled content before the delay expires.
  • The overflow: visible on .cm-dl-progress-track allows the glowing dot on progress fills to extend beyond the 2px track without being clipped.
  • The FAB button uses ::before and ::after pseudo-elements for its two concentric pulse rings. The rings are staggered by setting animation-delay: 1.5s on the second ring to create a natural breathing rhythm.
  • The selection counter uses font-variant-numeric: tabular-nums to ensure "01" and "04" occupy the same width, preventing layout shifts as numbers change.
