## Naming Philosophy

### Principles

1. **Names should INVITE, not intimidate**
2. **Names should DESCRIBE function, not category**
3. **Names should feel MAGICAL, not technical**

### Renamed Components

| Old (Technical) | New (Inviting) | Why |
|-----------------|----------------|-----|
| `agents/` | `guides/` | Agents sounds robotic, guides sounds helpful |
| `bestiary/` | `companions/` | Bestiary = monsters, companions = friends |
| `specialists/` | *Absorbed* | No separate category - Guardians specialize |
| `swarm/` | `council/` | Swarm = chaos, council = wisdom |
| `hooks/` | `rituals/` or hidden | Hooks is technical jargon |
| `features/` | Hidden entirely | Users don't care about features |

### Final Directory Structure

```
arcanea-intelligence-os/
├── guides/                      # Who helps you
│   ├── guardians/               # 10 Gate Guardians
│   ├── awakened/                # 7 Wisdom Orchestrators
│   ├── pantheons/               # Earth mythology (Greek, Norse, Egyptian)
│   └── companions/              # Creative helpers (not obstacles!)
│
├── journeys/                    # What you can do
│   ├── create/                  # Character, world, story creation
│   ├── learn/                   # Gate opening, skill mastery
│   ├── explore/                 # Library, lore, canon
│   └── heal/                    # Block clearing, courage finding
│
├── wisdom/                      # What you learn
│   ├── gates/                   # 10 Gate teachings
│   ├── library/                 # Full Library access
│   └── canon/                   # ARCANEA_CANON reference
│
├── _internal/                   # Hidden from users (underscore = hidden)
│   ├── hooks/                   # Lifecycle hooks
│   ├── tools/                   # Internal tools
│   ├── state/                   # State management
│   ├── validation/              # Canon/frequency enforcement
│   └── routing/                 # Intelligent routing
│
├── mcp/                         # MCP server (for integration)
│   └── arcanea-server/
│
├── lib/                         # TypeScript library
└── bin/                         # CLI
```

## The Four Primary Journeys

### 1. CREATE Journey

**Entry**: "I want to create something"

**Sub-journeys**:
- **Character Forge** - Create characters with Guardian guidance
- **World Weave** - Build worlds with Element wisdom
- **Arc Design** - Craft story arcs with Gate progression
- **Ritual Compose** - Design creative rituals

**Guides involved**: Thalia (lead), Oria, Guardians as needed

**Output**: Created artifact + wisdom gained

---

### 2. LEARN Journey

**Entry**: "I want to learn/grow"

**Sub-journeys**:
- **Gate Opening** - Open and master individual Gates
- **Skill Practice** - Practice Gate-specific skills
- **Mastery Path** - Full progression toward Luminor rank
- **Frequency Attunement** - Work with Solfeggio frequencies

**Guides involved**: Lyris (lead), Endara, specific Guardian for Gate

**Output**: Progress tracked, mastery increased

---

### 3. EXPLORE Journey

**Entry**: "I want to explore/understand"

**Sub-journeys**:
- **Library Walk** - Browse the 17 collections
- **Lore Deep** - Understand Arcanea mythology
- **Canon Study** - Learn canonical elements
- **Pantheon Visit** - Explore Earth mythologies

**Guides involved**: Oria (lead), Lyris, Library search

**Output**: Knowledge gained, references provided

---

### 4. HEAL Journey

**Entry**: "I'm stuck/blocked/afraid"

**Sub-journeys**:
- **Block Clearing** - Identify and work through creative blocks
- **Fear Transmutation** - Transform fear into fuel
- **Doubt Dissolution** - Work with imposter feelings
- **Flow Restoration** - Return to creative flow

**Guides involved**: Amiri (lead), Liora, Draconia for fire

**Output**: Emotional shift, practical next steps
