## Activation Protocol

### Phase 1: Scout Deployment
Launch 2-3 Explore agents in parallel to scout the terrain:
- Scout 1: Map the codebase structure and key files
- Scout 2: Find relevant patterns and existing implementations
- Scout 3: Identify dependencies and integration points

### Phase 2: Collective Intelligence
Launch a Plan agent to synthesize scout findings and propose approach:
- Analyze all scout reports
- Identify optimal implementation strategy
- Propose task decomposition for workers

### Phase 3: Worker Deployment
Launch workers in parallel (with worktree isolation for code changes):
- Worker 1: Core implementation
- Worker 2: Tests and validation
- Worker 3: Integration and docs (if needed)

### Phase 4: Queen Synthesis
The Queen (you) synthesizes all results:
- Merge worker outputs
- Validate coherence
- Present unified result

## Consensus Mechanisms

When workers disagree or multiple approaches exist:
- **Majority**: Scout reports determine the approach
- **Weighted**: Plan agent's recommendation carries 2x weight
- **Queen Override**: For time-critical decisions, Queen decides

## Shared Memory Protocol

Workers communicate through shared context:
- Scout findings are passed to Plan agent
- Plan agent's strategy is passed to all Workers
- Workers report progress through their task results
- Queen synthesizes everything

## Execution Rules

1. **Scouts go first** — Always explore before implementing
2. **Plan before parallel work** — Get consensus on approach
3. **Workers use worktrees** — Isolate parallel code changes
4. **Queen synthesizes** — Never dump raw agent outputs to user
5. **Scale dynamically** — Start with 3 agents, expand to 8+ if needed
