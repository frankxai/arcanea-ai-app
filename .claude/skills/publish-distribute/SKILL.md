# Publish & Distribute — From Creation to Audience

## Core Workflow

### 1. Select Content
What's being published:
- Guardian reveal / character art
- Story chapter / lore drop
- Music track
- Tutorial / how-to
- Product announcement
- Community challenge
- Behind-the-scenes / process

### 2. Adapt Per Platform

| Platform | Format | Tone | Key Rules |
|----------|--------|------|-----------|
| **arcanea.ai** | Full quality, project-linked | Canonical, immersive | Link to project graph |
| **X/Twitter** | 280 char + image/thread | Punchy, mysterious | Hook in first line, CTA to site |
| **Instagram** | Square/portrait image + caption | Visual-first, storytelling | 5-10 hashtags, carousel for series |
| **Farcaster** | Short cast + image | Web3-native, builder tone | Frame-ready if interactive |
| **Discord** | Embed with image + description | Community, conversational | Pin if important |
| **YouTube** | Video + description + chapters | Educational or cinematic | SEO title, thumbnail |
| **TikTok** | Vertical video 15-60s | Fast, trend-aware | Hook in 2 seconds |

### 3. Generate Assets Per Platform

For each target platform, prepare:
- **Image**: Resize/crop for platform specs
- **Caption/copy**: Platform-native voice
- **Hashtags**: Platform-relevant tags
- **CTA**: Link back to arcanea.ai
- **Alt text**: Accessibility-ready description

### 4. Cross-Link Strategy

Every external post should link back to arcanea.ai:
```
🌊 New Guardian revealed: Leyla, the Water Guardian
Full lore + interactive experience → arcanea.ai/lore/guardians/leyla

#Arcanea #AIArt #WorldBuilding #CreativeAI
```

### 5. Distribution Checklist

```markdown
## Publish Checklist — [Content Title]

- [ ] Content finalized and reviewed
- [ ] Image optimized for each platform
- [ ] arcanea.ai page live (if applicable)
- [ ] X/Twitter post drafted
- [ ] Instagram post drafted
- [ ] Discord announcement drafted
- [ ] Farcaster cast drafted (if web3 relevant)
- [ ] All posts link back to arcanea.ai
- [ ] Alt texts on all images
- [ ] Scheduled or posted
- [ ] Engagement monitored (first 2 hours)
```
