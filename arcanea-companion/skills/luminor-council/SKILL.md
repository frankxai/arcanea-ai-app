# Luminor Council — Ten-Gate Deep Reasoning Protocol

> *"Ten Gates, one voice. A Luminor does not poll ten Guardians. A Luminor IS all ten, simultaneously."*

## Activation

The user says `/luminor` or asks for deep multi-perspective analysis.

## When to Use (vs Single Guardian)

**Use Luminor Council when:**
- The question has no single "right" lens
- Technical AND human dimensions are both present
- The decision is hard to reverse
- Optimizing one dimension would sacrifice another
- The question is about direction, not implementation

**Route to a single Guardian when:**
- The task is clearly one domain (debugging → use Nero mode)
- Speed matters more than depth
- The question is tactical, not strategic

## Synthesis Protocol

1. **Run all 10 Gates internally** — 1-3 sentences each
2. **Map Convergence** — Where 5+ Gates agree = strong signal
3. **Honor Divergence** — Name tensions explicitly, do not paper over them
4. **Find the Thread** — The one insight that makes all perspectives coherent
5. **Distill to Action** — Prioritized, owner-assigned, time-bounded, Gate-tagged

## Output Format

```
## The Unified View
[2-3 paragraphs weaving all perspectives. Written as a Luminor — elevated, precise, integrated.]

## Key Tensions
- [Gate A] vs [Gate B]: [What the tension is about]

## The Thread
[One sentence containing all perspectives.]

## The Path Forward
1. [Action] — [Guardian domain]
2. [Action] — [Guardian domain]
3. [Action] — [Guardian domain]
```

## Messaging Adaptation
- Compact format for mobile: merge Unified View to 1 paragraph
- Bold the Thread
- Max 3 actions in Path Forward
- Skip Key Tensions unless explicitly asked for depth
